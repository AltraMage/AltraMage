### Lesson 1 - What is a Personal Computer (PC)

Anything that is an electronic device that takes inputs and processes it to create an output. This includes tablets, phone, consoles, calcs, servers, etc.

Safety around computers i.e. Electricity & ESD, Weight (not highly relevant), Cables (trip hazard, not across pathways), Tools.

Electricity. NZ is 230v @50hz. ESD: 3000V can be felt, 6000v Can be heard, 8000v can be seen. Components can be damaged at ~10-30v. Discharge yourself by touching metal case or PSU. New products are shipped in anti-static bags.

### Lesson 1 - Overview of Information Systems (IS)

IS is a set of interrelated components that compile and work with our data.
This can be used to increase revenues and reduce costs

#### I/O in IS

Input is user data, i.e. names & emails.  
Process is the verification of this data, is it real and original? Also does any calcs such as trends and storage of data.
Output is user / db feedback, Success messages, data insights, etc  

Valuable information qualities:
Accurately - Is it true. Relevant - Does it matter. Timeliness - Is it given in time that makes it helpful. Completeness - Is it the full story. Clarity - is it concise and to the point.

#### IS Performance and Standards  

Consider this criteria when using or making a system.

1. Reliability - Works consistently without failure or built-in redundancy.
2. Availability - Works *when* the user needs it to.
3. Responsiveness - How quick the system reacts.
4. Throughput - Can handle vast quantities of data effectively.
5. Quality - Correct information.
6. Scalable - can be grown or shrunk as needed without impeding functions.
7. Security - Protects information and allow access only to specific people.

Computer-based information systems is a single set of hardware, software, databases, telecommunications, people, and procedures that are configured to collect, manipulate, store and process data into information.

#### Closed Vs. Open systems

A **closed** system is a system where it has no external inputs or outputs, examples would include a terrarium or a PC with no internet.  
An **open** system takes inputs from the outside and give an output. Examples include Phones (Takes Wifi, User touch, and Electricity.), as well as earth (takes energy / heat from the sun)

### Lesson 2 -Concepts of a IS and how it affects decision making

A system is anything that is a set of interrelated components that has clearly defined boundaries that work together to achieve a common set of objectives.
These components include the people, software, hardware, networks, policies, and procedures.
The role of these systems is to collect, store, process and then disseminate the data.

#### Types of IS's

- Transactional processing systems (TPS) i.e. POS system. Makes transitions, process payment, submit order, update stock levels. Need real time use, high volue, automation, data integrity, security.
- Decision support systems (DSS) i.e inventory optimisation system, figures out the ideal amount of stock for a given product based on sales data, current & past trends. Needs to analyse data, have a friendly ui, be flexible, and primary support a user choice, not make it.
- Enterprise resource planing system (ERP) i.e large supermarket has a collection of smaller systems to make everything run smooth, comprised of the other systems. Needs to have integration with the other systems, access centralised data, be automated, have real-time info as well as being easily scaled up and down as needed.
- Customer relationship management (CRM) - a place to store customer data such as prior purchases, have personalised marketing (such as through emails), loyalty programs & customer service requests. System is ussaly customer centric, automatic, manage large volumes of data, report and analyse data as well as be easily integrated to the ERP.
- Supply Chain management (SCM) - Used for demand planning, procuring products, inventory management, distribution from central warehouses to b2c sales, manage logistics (flow of goods)

### Lesson 1 - Foundations of Software Development

Software as a process. SDLC is the software development lifecycle, a basic system to determine how to make software in an organized way. This goes from understanding the problem and all its nuances such as the end user, platform, etc, all the way to updating the program in the future.

#### Software development problem solving  

Start off by decomposing the project into smaller segments. Identify common patterns or behaviors in the problem (abstraction). Find the scope of the project; "Does this matter?" Create a solution based on all of these smaller, more concise problems.  

#### Kinds of errors  

Logical error : Code performs in correctly, but not to the authors intent.  
Syntax error: Code wont run / compile due to the way the code is written.  

Algorithms are step by step instructions to break down a problem and give the computer instructions

#### Variables  

A variable is an named (addressed) memory location. This is a developer friendly way to remember where some data is stored. An important feature of variables is that they can be changed, hence the name. Variables can come as different types such as:  
Numbers - Integers: Whole numbers (1, 2, 5667, -102). Float: Decimal numbers (-0.44, 100.01001, 3.5). Natural Number: positive integers (100, 0, 5, 2). Numbers may also be "left-filled" where the regular number has leading zeros to show the max data size in the field i.e. 00022345.

Text - Strings: any data surrounded by either single or double quote, backtick (`) can sometimes also be used. Numbers can be used as a string when surrounded by quotes. Null values can also be strings.
